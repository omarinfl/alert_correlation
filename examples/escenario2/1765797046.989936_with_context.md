# Incident Report: Unauthorized Execution of Audit Service and Promiscuous Mode Activation

## 1. Executive Summary
On 2025-12-15 at 11:10:46 UTC, the host `inetfw` (192.168.100.254) recorded a high-severity event involving the execution of `systemctl start auditf.service` with root privileges. This action immediately preceded the network interface `ens3` entering promiscuous mode. The sequence of events suggests a deliberate attempt to enable packet sniffing or network monitoring capabilities under the guise of a system service, indicating a potential compromise or unauthorized administrative activity.

## 2. Incident Context
The alert is part of a larger chain of suspicious activity. 
1. **Initial Access/Session:** A sudo session was opened by user `john` at 11:10:26, followed by a rapid closure.
2. **Service Execution:** At 11:10:45, user `john` again utilized `sudo` to execute `systemctl start auditf.service`.
3. **Malicious Outcome:** Immediately following the service start, the system logged an `ANOM_PROMISCUOUS` event, confirming that the `auditf` service triggered the network interface `ens3` to enter promiscuous mode. 
The `auditf` service appears to be a masqueraded process or a tool configured to perform unauthorized network traffic capture.

## 3. Threat Intelligence Correlation

### 3.1 Validated Evidence-Based Mapping (RAG)
*   **T1569.003 (Systemctl):** The alert explicitly captures the use of `systemctl` to start a service. This technique is used by adversaries to execute programs as system services.
*   **T1548.003 (Sudo and Sudo Caching):** The event was triggered via `sudo` elevation. The context shows multiple rapid sudo sessions, which is consistent with an adversary testing or utilizing elevated privileges.
*   **T1543.002 (Systemd Service):** The execution of `auditf.service` demonstrates the abuse of systemd unit files to execute code with root privileges, a common method for both persistence and privilege escalation.

### 3.2 AI Heuristic Proposals (Unverified)
*Warning: These are AI inferences based on contextual behavior and have NOT been validated.*
*   **T1040 (Network Sniffing):** The transition of the interface to promiscuous mode is the primary indicator of network sniffing. The `auditf` service is likely a wrapper for a packet capture tool (e.g., `tcpdump` or `tshark`).
*   **T1078.003 (Valid Accounts: Local Account):** The use of user `john`'s credentials to perform these actions suggests that the account may be compromised or that the user is performing unauthorized administrative tasks.

## 4. Assessment
**Severity: Critical**
The combination of privilege escalation (`sudo`), service manipulation (`systemctl`), and the activation of promiscuous mode on a network-facing host (`inetfw`) represents a significant security breach. The ability to sniff network traffic at the firewall/gateway level allows an attacker to intercept sensitive data, credentials, and internal network communications.

## 5. Recommended Actions
1.  **Immediate Containment:** Stop the `auditf.service` immediately (`systemctl stop auditf.service`) and disable it (`systemctl disable auditf.service`).
2.  **Network Isolation:** Isolate the host `inetfw` from the network to prevent further data exfiltration while the investigation is ongoing.
3.  **Forensic Analysis:**
    *   Inspect the contents of the `auditf.service` unit file (typically in `/etc/systemd/system/`) to identify the binary or script it executes.
    *   Examine the binary `/usr/bin/auditf` for malicious code or unexpected behavior.
    *   Review all logs for user `john` to determine if the account was used by an authorized administrator or an unauthorized actor.
4.  **Credential Rotation:** Force a password reset for user `john` and audit the `sudoers` file for any unauthorized entries or excessive permissions.
5.  **Persistence Check:** Search for other suspicious systemd services or cron jobs created around the same timestamp.
