# Incident Report: Automated Reconnaissance and SQL Injection Attempt against Web Server

## 1. Executive Summary
On September 22, 2025, the host `videoserver` (172.17.100.121) was targeted by an external actor (192.42.1.174) using the Nikto vulnerability scanner. The activity consisted of an automated reconnaissance phase followed by a targeted SQL injection attempt against a web application endpoint. This represents an active, automated attack attempt rather than an isolated anomaly.

## 2. Incident Context
The original alert (SQL injection attempt) is the culmination of a broader automated scanning campaign. The 'CONTEXT WINDOW' reveals that the source IP 192.42.1.174 performed rapid directory and file enumeration (e.g., `/servlet/SimpleServlet`, `/admcgi/contents.htm`) using the Nikto 2.1.5 scanner. The SQL injection attempt on `/xsql/demo/adhocsql/query.xsql` is a direct follow-up to this discovery phase, where the attacker attempted to extract database user information (`select username from ALL_USERS`). The 404 errors observed in the context indicate that the attacker is probing for common vulnerabilities and paths that may not exist on this specific server.

## 3. Threat Intelligence Correlation

### 3.1 Validated Evidence-Based Mapping (RAG)
*   **T1190 - Exploit Public-Facing Application**: The attacker attempted to inject SQL commands into a web application endpoint. This is a classic method for exploiting public-facing web services to gain unauthorized access or extract data.
*   **T1595.002 - Vulnerability Scanning**: The use of the "Nikto/2.1.5" user-agent string and the rapid sequence of requests targeting various paths confirms the attacker is performing automated vulnerability scanning to identify weaknesses.
*   **T1046 - Network Service Discovery**: The attacker is systematically probing for available services and directories on the web server, which is a precursor to identifying exploitable entry points.
*   **T1594 - Search Victim-Owned Websites**: The attacker is actively crawling the web server's directory structure to identify sensitive files or vulnerable scripts, aligning with the reconnaissance tactic of searching victim-owned assets.

### 3.2 AI Heuristic Proposals (Unverified)
*Warning: These are AI inferences based on contextual behavior and have NOT been validated.*
*   **T1505.003 - Server Software Component (Web Shell)**: While not yet confirmed, attackers using Nikto often look for paths to upload or interact with web shells. The probing of `/admcgi/` and `/servlet/` paths suggests the attacker is looking for execution environments where a web shell could be placed.

## 4. Assessment
**Severity: Medium**
The severity is rated as Medium because, while the attack is automated and persistent, there is no evidence of successful exploitation (the server returned 404s for the probes). However, the use of a known vulnerability scanner indicates a high intent to identify and exploit weaknesses. If the target application is vulnerable, the risk would escalate to Critical.

## 5. Recommended Actions
*   **Immediate Containment**: Block the source IP `192.42.1.174` at the perimeter firewall or Web Application Firewall (WAF).
*   **Investigation**: Review web server access logs for any 200 OK responses from the source IP to determine if any probes were successful.
*   **Vulnerability Management**: Audit the `/xsql/` directory and any associated applications for SQL injection vulnerabilities. Ensure all web applications are patched and follow secure coding practices (e.g., parameterized queries).
*   **Hardening**: Implement a WAF to automatically drop traffic from known vulnerability scanners and block suspicious request patterns (e.g., SQL keywords in URI parameters).
*   **Monitoring**: Increase logging verbosity for the web server to capture full request bodies for any future suspicious activity.
