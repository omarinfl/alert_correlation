# Incident Report: Unauthorized Execution of ZoneMinder Control Scripts via Sudo

## 1. Executive Summary
This incident involves the execution of administrative commands (`zmdc.pl check` and `zmdc.pl startup`) on the host `videoserver` (172.17.100.121). The commands were executed by the `root` user acting as the `www-data` service account via `sudo`. While these commands are associated with the ZoneMinder application, the rapid succession of login sessions and command executions suggests either automated service management or potential misuse of the `sudo` mechanism. This is currently classified as an anomaly requiring verification of administrative intent.

## 2. Incident Context
The original alert identifies a single `sudo` execution of `/usr/bin/zmdc.pl check`. The context window reveals a broader pattern of activity occurring at `2025-09-22 18:53:21`:
*   Multiple rapid `PAM` login sessions were opened and closed for the `www-data` user.
*   A secondary command, `/usr/bin/zmdc.pl startup`, was executed immediately following the `check` command.
*   The sequence indicates that the `root` user is actively managing the ZoneMinder service by invoking scripts as the `www-data` user. This behavior is consistent with service orchestration but warrants confirmation to ensure it was not triggered by an unauthorized actor or a compromised web application process.

## 3. Threat Intelligence Correlation

### 3.1 Validated Evidence-Based Mapping (RAG)
*   **T1548.003 - Sudo and Sudo Caching:** The alert explicitly triggers on `sudo` execution. The use of `sudo` to run commands as `www-data` is the core mechanism of this technique. The rapid session opening/closing suggests potential abuse of sudoers permissions to spawn processes.
*   **T1548 - Abuse Elevation Control Mechanism:** As the parent technique for sudo-based elevation, this applies as the system is utilizing `sudo` to switch user contexts to `www-data`.
*   **T1059.004 - Unix Shell:** The execution of the Perl script `zmdc.pl` via the shell environment falls under the abuse of Unix shell commands for execution.

### 3.2 AI Heuristic Proposals (Unverified)
*Warning: These are AI inferences based on contextual behavior and have NOT been validated.*
*   **T1078.002 - Valid Accounts (Default Accounts):** If the `www-data` account is being accessed by an unauthorized party to execute scripts, this could represent the abuse of a service account to maintain persistence or perform lateral movement within the web directory.

## 4. Assessment
**Severity: Medium**
The activity is technically authorized (as it is being performed by `root`), but the rapid, repetitive nature of the `PAM` sessions and the execution of service control scripts (`check` and `startup`) is suspicious. If this was not a scheduled maintenance task or a known automated script, it could indicate an attempt to restart or manipulate the ZoneMinder service to exploit known vulnerabilities or bypass security controls.

## 5. Recommended Actions
1.  **Verify Administrative Intent:** Consult with the system administration team to determine if any automated scripts or maintenance tasks were scheduled for `videoserver` at `18:53:21`.
2.  **Audit Sudoers Configuration:** Review the `/etc/sudoers` file to ensure that the `root` user (or other users) is restricted to only the necessary commands for `www-data` and that `NOPASSWD` is not overly permissive.
3.  **Review ZoneMinder Logs:** Examine the ZoneMinder application logs (`/var/log/zm/`) for any errors or unusual activity coinciding with the `zmdc.pl` execution.
4.  **Implement TTY Tickets:** Enable `tty_tickets` in the sudo configuration to prevent privilege leakage across different terminal sessions.
5.  **Monitor for Persistence:** Check for any unauthorized modifications to the ZoneMinder configuration files or startup scripts that might have been made during this window.
