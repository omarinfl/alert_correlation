# Incident Report: Unauthorized Creation of Local User Account on 'videoserver'

## 1. Executive Summary
A new local user account, `webadmin`, was created on the host `videoserver` (172.17.100.121) at 18:56:00 UTC. This action was performed via the `useradd` utility. Given the context of recent `sudo` activity by the `www-data` service account, this event is highly suspicious and suggests a potential persistence mechanism being established by an unauthorized actor.

## 2. Incident Context
The alert is part of a sequence of events indicating suspicious system modification. 
- **18:53:24 UTC:** A `sudo` session was opened for the `www-data` service account. Service accounts (like `www-data`) should typically not be performing interactive `sudo` operations.
- **18:56:00 UTC:** Immediately following the suspicious `sudo` activity, the `useradd` command was executed to create a new user and group named `webadmin`. 
The creation of a new user account by a service account is a significant security anomaly, suggesting that the `www-data` account may have been compromised and used to escalate privileges or establish persistence.

## 3. Threat Intelligence Correlation

### 3.1 Validated Evidence-Based Mapping (RAG)
*   **Technique ID: T1136.001 (Local Account)**
    *   **Tactic:** Persistence
    *   **Description:** Adversaries may create a local account to maintain access to victim systems.

*   **Technique ID: T1136 (Create Account)**
    *   **Tactic:** Persistence
    *   **Description:** Adversaries may create an account to maintain access to victim systems.
    *   **Rationale:** This is the parent technique for T1136.001 and describes the general behavior of creating an account to maintain persistence.

*   **T1068 (Exploitation for Privilege Escalation):** The transition from a web service account (`www-data`) to executing administrative commands (`useradd`) strongly suggests that an exploit or misconfiguration was used to escalate privileges.

## 4. Assessment
**Severity: High**
The severity is rated as High because the creation of a new user account is a classic indicator of persistence establishment. The fact that this was preceded by an unusual `sudo` session from the `www-data` service account indicates a high probability of a successful compromise of the web server environment.

## 5. Recommended Actions
1.  **Immediate Containment:** Disable the `webadmin` account immediately (`usermod -L webadmin`) and lock the account.
2.  **Session Termination:** Terminate any active sessions associated with the `www-data` user and investigate the web application logs for signs of remote code execution (RCE) or web shell activity.
3.  **Credential Rotation:** Rotate credentials for all service accounts and administrative users on `videoserver`.
4.  **Forensic Investigation:** Inspect `/var/log/auth.log` and web server access logs (e.g., Apache/Nginx) for the source IP address that initiated the `sudo` session at 18:53:24.
5.  **Hardening:** Review `sudoers` configuration to ensure that the `www-data` user does not have unnecessary privileges to execute administrative binaries like `useradd`. Implement Multi-Factor Authentication (MFA) for all administrative access.
