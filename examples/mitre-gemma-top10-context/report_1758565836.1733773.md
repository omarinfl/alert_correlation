# Incident Report: Automated Vulnerability Scanning via Nikto on Video Server

## 1. Executive Summary
A high-confidence reconnaissance event was detected targeting the `videoserver` (172.17.100.121). An external actor using the `Nikto/2.1.5` web vulnerability scanner attempted to access multiple sensitive file extensions (e.g., `.bak`, `.password`, `.php3`) to identify misconfigurations or exposed sensitive data. While the immediate alerts show a series of `404 Not Found` errors—indicating the specific files requested do not exist—this represents a targeted automated reconnaissance phase of a potential attack.

## 2. Incident Context
The primary alert identifies a suspicious `GET` request for `/borLjZ2z.bak`. Analysis of the **CONTEXT WINDOW** reveals that this is not an isolated anomaly but part of a rapid-fire automated scanning sequence. Within a single second, the source IP `192.42.1.174` generated multiple requests for various file extensions including `.password`, `.pl|dir`, `.php3`, and `.apw`. 

The sequence of events shows a classic "wordlist scanning" pattern where a tool iterates through a list of common sensitive file names and extensions to find unprotected backups, configuration files, or credentials. The high frequency of `404` error codes confirms the scanner is probing for non-existent resources to map the server's structure and potential vulnerabilities.

## 4. Assessment
**Severity: Medium**

**Justification:** 
The activity is a confirmed automated reconnaissance attempt using a known tool (Nikto). While the current logs show only `404` errors (indicating the attacker has not yet successfully accessed any sensitive files), the intent is clearly malicious. The severity is rated as **Medium** because, while the attack is in the reconnaissance phase and has not yet achieved a breach, the target is a public-facing `videoserver`, making it a high-value target for initial access attempts.

## 5. Recommended Actions
*   **IP Blocking:** Immediately block the source IP `192.42.1.174` at the edge firewall or WAF to terminate the current scanning session.
*   **WAF Implementation:** Ensure a Web Application Firewall (WAF) is active and configured with rules to detect and block common vulnerability scanners (e.g., Nikto, Acunetix, Nessus).
*   **Hardening & Patching:** 
    *   Verify that no sensitive files (e.g., `.bak`, `.sql`, `.password`, `.env`) are present in the web root.
    *   Ensure the web server software and all hosted applications are fully patched against known vulnerabilities.
*   **Log Monitoring:** Monitor for any `200 OK` responses from the same source IP or similar patterns to ensure the scanner did not successfully find a valid resource.
*   **Information Minimization:** As per MITRE mitigation for T1595, ensure that any non-essential services or directories are not exposed to the public internet.
