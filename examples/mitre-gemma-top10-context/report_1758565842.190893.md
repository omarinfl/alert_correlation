# Incident Report: Automated Web Vulnerability Scanning (Nikto) Targeting Path Traversal

## 1. Executive Summary
A high-intensity automated web vulnerability scan was detected targeting the `videoserver` (172.17.100.121). The attack originated from source IP `192.42.1.174` and utilized the **Nikto/2.1.5** scanner to attempt Path Traversal and Local File Inclusion (LFI) attacks. The primary objective appeared to be the unauthorized access of sensitive system files, specifically `/etc/passwd`. While the logs indicate a `404 Not Found` response for the specific payloads, the volume and nature of the requests suggest a targeted reconnaissance phase by an external adversary.

## 2. Incident Context
The incident is characterized by a rapid succession of malicious HTTP GET requests. The 'Original Alert' captures a specific attempt to access `/cgi-bin/pollit/Poll_It_SSI_v2.0.cgi` with a payload designed to traverse directories and read the `/etc/passwd` file (`data_dir=\\etc\\passwd%00`). 

The **CONTEXT WINDOW** reveals that this was not an isolated event but part of a broader, automated scanning campaign. Multiple requests (over 300 instances of 400-level errors) were observed in a very short timeframe, targeting various CGI scripts (e.g., `html2chtml.cgi`, `html2wml.cgi`, `echo.bat`) with different injection patterns. The high frequency of these requests confirms that the main alert is a symptom of an active, automated vulnerability scan (Nikto) attempting to identify exploitable web interfaces and directory traversal vulnerabilities on the host.

## 3. Threat Intelligence Correlation

### 3.1 Validated Evidence-Based Mapping (RAG)
The following techniques and vulnerabilities have been explicitly matched through the correlation engine:

*   **T1003.008 - /etc/passwd and /etc/shadow (Credential Access):** The attacker specifically targeted the `/etc/passwd` file via a directory traversal payload. This is a known technique used to dump user account information for offline cracking.
*   **T1190 - Exploit Public-Facing Application (Initial Access):** The use of the Nikto scanner to target CGI scripts on a web server is a direct attempt to exploit potential weaknesses in an internet-facing application to gain initial access.
*   **T1087 & T1087.001 - Account Discovery / Local Account (Discovery):** The attempt to read `/etc/passwd` serves the purpose of enumerating local system accounts to aid in subsequent stages of an attack.
*   **T1203 - Exploitation for Client Execution (Execution):** The delivery of crafted parameters to a CGI script is a common method used to achieve arbitrary code execution.
*   **CVE-2012-1823 (PHP-CGI Query String Parameter Vulnerability):** The request structure (manipulating query strings in a CGI environment) is highly consistent with this vulnerability, which allows remote code execution.
*   **CVE-2020-3452 (Cisco ASA/FTD Path Traversal):** The use of directory traversal sequences (`\\etc\\passwd%00`) directly correlates with the behavior described in this vulnerability.
*   **CVE-2017-17562 (Embedthis GoAhead RCE):** The targeting of a CGI script with malicious parameters to access system files aligns with the exploitation patterns of this vulnerability.

## 4. Assessment
**Severity: High**

**Justification:** 
The severity is rated as **High** because the activity represents an active, automated attempt to exploit the server. The attacker is specifically targeting sensitive system files (`/etc/passwd`) and utilizing a known vulnerability scanner (Nikto). While the current logs show `404` errors (suggesting the specific files/paths might not exist or the attack failed), the intent is clearly malicious and aimed at gaining unauthorized access or information disclosure. The presence of multiple high-severity CVE correlations (including a CVSS 9.8 vulnerability) further justifies this rating.

## 5. Recommended Actions
1.  **Immediate IP Blocking:** Implement a temporary block on the source IP `192.42.1.174` at the perimeter firewall/WAF to halt the ongoing scan.
2.  **Verify File Integrity:** Conduct a file integrity check on the `/etc/` directory and the `/cgi-bin/` directory to ensure no unauthorized changes or backdoors were planted.
3.  **Patch Management:** Review the versions of all CGI scripts and web-facing applications (specifically those related to `Poll_It_SSI` or similar) and ensure they are patched against the identified CVEs (CVE-2012-1823, CVE-2020-3452, etc.).
4.  **WAF Rule Optimization:** Ensure the Web Application Firewall (WAF) has active rules to detect and block Path Traversal (`../`, `\etc\`) and common CGI injection patterns.
5.  **Log Review:** Perform a deep dive into the web server logs for the past 24 hours to see if any requests resulted in `200 OK` status codes, which would indicate a successful exploitation.
