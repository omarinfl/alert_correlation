# Incident Report: Automated Vulnerability Scanning via Nikto (IP: 192.42.1.174)

## 1. Executive Summary
A high-severity automated web vulnerability scan was detected targeting the `videoserver` (172.17.100.121). The attack originated from source IP `192.42.1.174` and utilized the **Nikto (v2.1.5)** scanning tool. The attacker attempted to identify exploitable vulnerabilities through directory traversal, command injection, and unauthorized access to sensitive system files. While the current logs show a high frequency of `404 Not Found` errors—indicating the specific paths requested do not exist on this host—the activity represents a clear reconnaissance phase of a targeted attack.

## 2. Incident Context
The `Original Alert` identifies a high-frequency event (Level 10) where multiple common web attacks were detected from a single source. The `CONTEXT WINDOW` confirms a rapid succession of requests (within seconds) characterized by the User-Agent string `Mozilla/5.00 (Nikto/2.1.5)`. 

The sequence of events shows the attacker systematically testing various URI paths to find weaknesses, including:
* **Command Injection attempts:** `/cgi-bin/cmd.exe?/c+dir` and `/scripts/root.exe`.
* **Directory Traversal/Sensitive File Access:** `../../database/rbsserv.mdb` and `/etc/passwd`.
* **Configuration Probing:** `abyss.conf` and `ServerInfo.jsp`.

The "Multiple common web attacks" alert is the direct result of this automated scanning behavior. The high volume of `404` errors suggests the scanner is currently in a "brute-force" discovery mode, attempting to map the attack surface of the `videoserver`.

## 4. Assessment
**Severity: High**

**Justification:** 
The activity is a highly structured, automated reconnaissance mission using a known exploitation tool (Nikto). While the current results are `404` (unsuccessful), the attacker is specifically targeting high-value assets such as system credentials (`/etc/passwd`) and command execution (`cmd.exe`). This represents a significant threat to the `videoserver` and serves as a precursor to a full compromise.

## 5. Recommended Actions
* **Immediate Containment:** Block the source IP `192.42.1.174` at the perimeter firewall/WAF to halt the current scanning activity.
* **Vulnerability Management:** Perform a formal, authorized vulnerability scan on the `videoserver` (172.17.100.121) to ensure no actual vulnerabilities (like the ones being probed) exist.
* **Web Application Firewall (WAF) Tuning:** Implement or update WAF rules to specifically detect and block common patterns found in the Nikto User-Agent and directory traversal attempts (e.g., `../`, `%00`, and `cmd.exe`).
* **Hardening:** 
    * Ensure the web server process runs with the least privilege (not as root).
    * Disable or remove unnecessary CGI scripts and directories (e.g., `/cgi-bin/`) if they are not required for business operations.
* **Monitoring:** Increase logging verbosity for the `videoserver` and monitor for any successful `200 OK` responses following similar patterns from other IP addresses.
