# Incident Report: Reconnaissance and Directory Discovery Attempt on 'videoserver'

## 1. Executive Summary
A high-confidence reconnaissance event was detected targeting the `videoserver` (172.17.100.121). An external actor (IP: 192.42.1.174) attempted to access forbidden directories and sensitive system files, specifically targeting `/var/www/default/log/` and `/etc/passwd`. The activity represents a targeted attempt at information gathering and directory traversal, likely aimed at identifying misconfigurations or sensitive log files to facilitate further exploitation.

## 2. Incident Context
The primary alert (Apache AH01276) indicates a failed attempt to access a directory index in `/var/www/default/log/`. While the Apache server successfully blocked the request due to the `Options` directive, the **CONTEXT WINDOW** reveals this is not an isolated error but part of a sustained, multi-stage attack pattern.

The sequence of events shows the attacker (192.42.1.174) performing the following:
1.  **Path Traversal/Sensitive File Probing:** Multiple Suricata alerts triggered for `/etc/passwd` detected in the URI, indicating an attempt to read system user files.
2.  **Information Gathering:** An attempt to access `phpinfo` to leak server configuration details.
3.  **Access Denial:** An attempt to access a `.htaccess` file in `/usr/lib/cgi-bin/`, which was denied by server configuration.
4.  **Directory Enumeration:** The main alert confirms the attempt to list the contents of the `/log/` directory.

The Apache error is a direct operational symptom of an active web-based reconnaissance campaign.

## 4. Assessment
**Severity: High**

**Justification:**
The severity is rated as **High** because the logs demonstrate a clear, multi-vector reconnaissance pattern. The attacker is not just hitting a single error but is actively probing for system-level files (`/etc/passwd`), configuration files (`phpinfo`, `.htaccess`), and directory structures. While the Apache server's security directives successfully prevented the directory listing, the intent is clearly malicious and indicates a highly motivated actor performing pre-exploitation activities.

## 5. Recommended Actions
*   **Immediate Contain-ment:** Block the source IP address `192.42.1.174` at the perimeter firewall/WAF.
*   **Vulnerability Assessment:** Perform a full vulnerability scan on the `videoserver` to ensure no other public-facing applications or scripts are vulnerable to Path Traversal or Local File Inclusion (LFI).
*   **Hardening:** 
    *   Ensure `AllowOverride None` is set for sensitive directories to prevent `.htaccess` manipulation.
    *   Verify that the web server user has minimal permissions (Least Privilege) and cannot access `/etc/` or other system directories.
*   **Log Review:** Audit web server access logs for any `200 OK` responses to similar requests to ensure no previous attempts were successful.
*   **WAF Tuning:** Implement or update Web Application Firewall (WAF) rules to automatically block requests containing strings like `/etc/passwd` or `phpinfo`.
